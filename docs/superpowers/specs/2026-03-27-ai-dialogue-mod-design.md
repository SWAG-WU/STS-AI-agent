# 杀戮尖塔2 AI对话Mod 设计文档

## 概述

为杀戮尖塔2（Slay the Spire 2）制作一个mod，在所有非火堆事件中注入一个"对话"按钮，玩家可以主动触发与怪物/NPC的AI对话。对话过程中每轮实时产生游戏效果，由AI根据随机生成的角色性格自由裁决结果。

## 技术栈

- **游戏引擎**: Godot 4.5（MegaDot）
- **语言**: C# / .NET 9.0
- **Mod API**: MegaCrit.Sts2.Core.Modding（[ModInitializer]）
- **运行时补丁**: HarmonyLib（游戏内置 0Harmony.dll）
- **AI后端**: 在线LLM API（Claude/GPT等）
- **语言支持**: 中英双语，玩家可在设置中切换

### 核心模块

1. **UIInjector** — HarmonyLib Postfix patch，在各UI加载完成后注入"对话"按钮
2. **PersonalityGenerator** — 根据事件类型随机生成2个性格组合
3. **DialoguePanel** — Godot CanvasLayer 动态创建的对话面板
4. **AIService** — prompt组装、API调用、JSON解析
5. **ActionExecutor** — 将AI返回的结构化指令转化为游戏内实际操作

### 核心设计原则

- **玩家主动触发** — 按钮注入，不拦截原有逻辑，不点就不触发
- **稳定第一** — 全链路try-catch，任何环节失败回退到原版游戏逻辑

## 2. 性格系统

每次触发对话时，为怪物/NPC随机生成**2个性格**。

### 性格池

| 性格 | 类型 | 概率权重 | 典型表现 |
|------|------|---------|---------|
| 暴躁 | 负面 | 高 (~15%) | 容易被激怒，动手快 |
| 狡诈 | 负面 | 高 (~15%) | 识破谎言，反套路玩家 |
| 贪财 | 中性 | 高 (~15%) | 可被贿赂，但开价也高 |
| 冷静 | 中性 | 中 (~13%) | 不受情绪影响，理性判断 |
| 胆小 | 正面 | 中 (~12%) | 容易被吓退，可能主动让路 |
| 怯战 | 正面 | 低 (~8%) | 害怕冲突，倾向和平解决 |
| 软弱 | 正面 | 低 (~7%) | 容易屈服于威胁/施压 |
| 善良 | 正面 | 低 (~5%) | 主动帮助玩家，给予好处 |
| 温和 | 正面 | 低 (~5%) | 耐心倾听，容易被说服 |
| 慷慨 | 正面 | 极低 (~3%) | 主动送礼物、降价、给物品 |

> 概率数字为近似值，两个性格独立抽取，可以组合出有趣的矛盾（如"贪财+善良"）。

### 角色类型的性格限制

| 角色类型 | 规则 |
|---------|------|
| 普通怪 | 完全随机 |
| 精英怪 | 偏向"冷静/聪明"，正面性格权重减半 |
| Boss | 固定一个"冷静"或"狡诈"，第二个随机（善良/慷慨/软弱权重再降半） |
| 商人 | 固定"贪财"，第二个随机 |
| 随机事件NPC | 完全随机 |

### 性格展示

性格信息不直接展示给玩家。AI的对话风格会自然流露性格特征，玩家在对话过程中自行判断。

## 3. 对话流程

### 触发方式

| 场景 | 按钮位置 |
|------|---------|
| 战斗中 | "结束回合"按钮旁边 |
| 商店 | 商店界面顶部或底部 |
| 随机事件(?) | 事件选项列表底部追加"尝试对话..."选项 |
| 精英/Boss战斗 | 同普通战斗 |
| 火堆 | **不注入**（RestSiteFilter排除） |

### 对话规则

- 最多**5轮**对话（玩家发言5次）
- **每轮实时生效** — AI每次回复都可附带游戏动作，立刻执行
- AI可以在任何轮次设置 `end_conversation: true` 提前结束
- 玩家可随时点"放弃"退出对话，回到原版事件逻辑

### 对话开场

玩家点击"对话"按钮后，AI先根据性格生成一句开场白（第0轮，不计入5轮限制），然后玩家再回复。开场白可以附带action（如怪物一见面就威胁性地挥了一拳）。

### 单轮流程

```
玩家输入文字 → 发送给AI（附带完整上下文）
    → AI返回JSON（dialogue + actions）
        → 对话框显示dialogue文本（怪物侧，靠右）
        → ActionExecutor执行actions（实时生效）
        → 显示⚡效果提示
        → 等待玩家下一轮输入（或对话结束）
```

### 对话流程示例

```
第1轮：
  [左·玩家] "大哥饶命啊！"
                        [右·怪物] "哼，又一个不自量力的冒险者"
                        ⚡ 无效果

第2轮：
  [左·玩家] "我可以出100金币买路"
                        [右·怪物] "就这点？你在侮辱我？！"
                        ⚡ 怪物攻击力+3（它看起来不耐烦了）

第3轮：
  [左·玩家] "对不起！150！150够吗？"
                        [右·怪物] "嗯...算你识相"
                        ⚡ 玩家-150金币，怪物攻击力恢复正常

第4轮：
  [左·玩家] "那我可以走了吗？"
                        [右·怪物] "滚吧，别让我再看到你"
                        ⚡ 跳过战斗！（对话结束）
```

## 4. 动作系统

### AI回复格式

AI每轮返回结构化JSON（玩家不可见，只看到dialogue文本）：

```json
{
  "dialogue": "哼，又一个不自量力的冒险者",
  "emotion": "threatening",
  "actions": [
    {"type": "modify_enemy_strength", "value": 3}
  ],
  "end_conversation": false
}
```

### 完整动作清单

#### 数值类

| 动作 | 参数 | 说明 |
|------|------|------|
| `modify_player_hp` | value (±整数) | 加减玩家血量 |
| `modify_player_gold` | value (±整数) | 加减金币 |
| `modify_enemy_strength` | value (±整数) | 加减怪物攻击力 |
| `modify_enemy_hp` | value (±整数) | 加减怪物血量 |

#### 增益/减益类（带时效）

| 动作 | 参数 | 说明 |
|------|------|------|
| `add_player_buff` | buff_id, stacks, duration | 给玩家加增益 |
| `add_player_debuff` | debuff_id, stacks, duration | 给玩家加减益 |
| `add_enemy_buff` | buff_id, stacks, duration | 给怪物加增益 |
| `add_enemy_debuff` | debuff_id, stacks, duration | 给怪物加减益 |

**duration 取值：**

| 值 | 含义 | 触发场景 |
|----|------|---------|
| `"combat"` | 仅本场战斗生效（默认值） | "公平竞技"、临时效果 |
| `"permanent"` | 永久生效 | 赠送的礼物、祝福 |
| `"turns:N"` | 持续N个回合 | "给你3回合逃跑时间" |

#### 卡牌类

| 动作 | 参数 | 说明 | 限制 |
|------|------|------|------|
| `give_card` | card_id | 赠送卡牌 | — |
| `destroy_card` | card_id 或 "random" | **永久销毁**卡牌（不可逆） | 仅"暴躁"或"狡诈"性格，概率再降50% |
| `steal_card` | card_id 或 "random" | **暂时夺取**卡牌（事件结束归还） | — |
| `return_card` | card_id | 归还之前被steal的卡牌 | — |

**steal_card机制：**
- 被偷的卡从牌组暂时移除，存入 `stolen_cards` 临时列表
- 当前事件/战斗结束时自动归还
- AI被说服后可提前用 `return_card` 归还

#### 物品类

| 动作 | 参数 | 说明 | 限制 |
|------|------|------|------|
| `give_relic` | relic_id | 赠送遗物 | 仅"慷慨"性格可触发 |

#### 事件控制类

| 动作 | 参数 | 说明 | 限制 |
|------|------|------|------|
| `skip_event` | — | 跳过当前事件/战斗 | 仅在 `end_conversation: true` 时可用 |
| `shop_discount` | percentage (0-100) | 商店全场打折 | — |
| `no_action` | — | 本轮无效果 | — |

### 安全限制

- 血量不能降到0以下（自动钳位到1HP）
- 金币不能为负（余额不足则不执行扣除）
- 每轮最多执行**2个动作**
- `skip_event` 仅在对话结束时可用
- `give_relic` 仅"慷慨"性格可触发
- `destroy_card` 仅"暴躁"或"狡诈"性格可触发
- `duration: "permanent"` 需要叙事合理性（AI prompt中约束）
- 未识别的动作自动忽略，记日志

## 5. AI Prompt设计

### 系统prompt模板

```
你是杀戮尖塔2中的一个{角色类型}：{角色名称}。

【你的性格】
{性格1}：{性格描述}
{性格2}：{性格描述}

【当前场景】
- 事件类型：{战斗/商店/随机事件/精英/Boss}
- 玩家当前状态：{HP}/{MaxHP}，金币{gold}
- 你的状态：{怪物HP/商品列表/事件描述}

【对话规则】
1. 始终保持角色扮演，用符合性格的语气说话，不要有固定句式
2. 每轮回复必须严格使用JSON格式：
   {"dialogue":"你的台词","emotion":"情绪","actions":[...],"end_conversation":false}
3. actions只能从以下清单中选择：[完整动作清单]
4. 每轮最多2个action
5. duration默认为"combat"，只有赠送礼物等叙事合理时才用"permanent"
6. destroy_card极少使用，仅在叙事中有强烈的不可逆理由时
7. 可在任何时候设置end_conversation:true结束对话
8. 对话最多5轮，第5轮必须设置end_conversation:true

【语言】
用{中文/English}回复
```

### AI prompt约束原则

- 不限制AI的说话句式，只限定性格和角色背景
- AI自由发挥对话内容，保证每次对话体验不同
- 通过性格描述间接影响AI的行为倾向

### 容错处理

| 异常情况 | 处理方式 |
|---------|---------|
| AI返回非法JSON | 尝试正则提取JSON部分，失败则显示纯文本 + no_action |
| action不在清单中 | 忽略该action，只执行合法的 |
| actions超过2个 | 只取前2个 |
| API超时（10秒） | 显示"（对方沉默了一会...）" + no_action |
| API返回错误 | 同超时处理 |

**布局规则：**
- 玩家消息：靠左对齐
- 怪物/NPC消息：靠右对齐（匹配游戏中怪物站位）
- 实时效果提示：居中，醒目颜色（黄/橙）
- 面板半透明覆盖在游戏画面上

**技术实现：**
- 使用 Godot `CanvasLayer` + `Control` 节点树动态创建
- 不修改任何原有Scene文件
- 主题资源 `dialogue_theme.tres` 控制字体、颜色、面板样式

### 设置界面

首次使用或未配置API Key时弹出配置面板：
- API Provider选择（Claude / GPT / 自定义）
- API Key输入
- API URL（自定义端点）
- 语言选择（中文/English）
- 模型选择

配置保存到 `%AppData%/AIDialogueMod/config.json`。

## 8. 错误处理与稳定性

### 全局策略

```
任何模块异常 → try-catch捕获 → 记录日志 → 回退原版逻辑 → 游戏继续
```

### 具体策略

| 场景 | 处理 |
|------|------|
| Harmony patch异常 | catch后返回，不阻止原方法执行 |
| API调用失败 | 显示"对方沉默了..."，no_action |
| API超时（10秒） | 同上 |
| JSON解析失败 | 显示AI原始文本作为对话，no_action |
| 未配置API Key | 弹出配置提示，按钮不可用 |
| 对话面板创建失败 | 按钮变灰不可点击 |
| 动作执行失败 | 跳过该动作，记日志，对话继续 |
| 配置文件损坏 | 使用默认值 |

### 打包产物

- `AIDialogueMod.dll` — 编译后的C#程序集
- `AIDialogueMod.pck` — Godot资源包（mod_manifest.json + 主题资源）
- 放入游戏目录 `mods/AIDialogueMod/` 即可加载
